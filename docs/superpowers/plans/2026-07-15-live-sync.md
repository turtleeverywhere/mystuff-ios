# Live Sync (Real-Time Listeners) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make shared items/locations (and one's own edits across devices) appear in real time instead of only after an app relaunch, and make friends/friend-requests + the account badge update live too.

**Architecture:** The data + social layers gain `AsyncStream`-based observe methods backed by Firestore `addSnapshotListener` (Mock yields once). `StuffViewModel` and `SocialViewModel` each start long-lived consumer tasks after their initial one-time load, updating observable state on every snapshot; tasks are cancelled on teardown (which removes the listeners). The existing one-time `loadData()`/`load()` (cache hydrate + backfill + migration-persist) is unchanged; the streams handle ongoing updates.

**Tech Stack:** Swift 6 / SwiftUI, Firebase Firestore snapshot listeners, AsyncStream.

## Global Constraints

- iOS 26.0, Swift 6.0 (strict concurrency). `@Observable` / `@MainActor`. Firestore uses Codable (`data(as:)`).
- **No test target exists.** Verification = compile via `xcodebuild` + deferred on-device gates. Build (repo root, look for `** BUILD SUCCEEDED **`):
  ```
  xcodebuild -project MyStuff.xcodeproj -scheme MyStuff -destination 'generic/platform=iOS Simulator' -configuration Debug build
  ```
- Reuse the exact existing queries: items/locations via `collectionGroup(...).whereField("memberIds", arrayContains: uid).order(by:"createdAt", descending:true)`; categories via `users/{uid}/categories`; friends via `users/{uid}/friends`; requests via `friendRequests where toUid/fromUid == uid`. Do NOT change rules/indexes (already deployed).
- Streams use `AsyncStream(bufferingPolicy: .bufferingNewest(1))` (only the latest snapshot matters — each yield replaces the whole array). On termination, `listener.remove()`.
- The stream consumers apply the SAME display mapping the one-time load uses (`migratedPhotoFields` / `migratedSharingFields` for items; `migratedSharingFields` for locations) but do NOT persist migrations (that stays a one-time concern in `loadData`).
- New Swift files (none expected) go under `MyStuff/`. Existing services are `@unchecked Sendable`; `[Item]`/`[Location]`/`[Category]`/`[Friend]`/`[FriendRequest]` are all `Sendable`.

```swift
    // MARK: - Live streams

    func itemsStream() -> AsyncStream<[Item]> {
        let query = db.collectionGroup("items")
            .whereField("memberIds", arrayContains: uid)
            .order(by: "createdAt", descending: true)
        return AsyncStream(bufferingPolicy: .bufferingNewest(1)) { continuation in
            let listener = query.addSnapshotListener { snapshot, _ in
                guard let snapshot else { return }
                continuation.yield(snapshot.documents.compactMap { try? $0.data(as: Item.self) })
            }
            continuation.onTermination = { _ in listener.remove() }
        }
    }

    func locationsStream() -> AsyncStream<[Location]> {
        let query = db.collectionGroup("locations")
            .whereField("memberIds", arrayContains: uid)
            .order(by: "createdAt", descending: true)
        return AsyncStream(bufferingPolicy: .bufferingNewest(1)) { continuation in
            let listener = query.addSnapshotListener { snapshot, _ in
                guard let snapshot else { return }
                continuation.yield(snapshot.documents.compactMap { try? $0.data(as: Location.self) })
            }
            continuation.onTermination = { _ in listener.remove() }
        }
    }

    func categoriesStream() -> AsyncStream<[Category]> {
        let query = categoriesCollection.order(by: "createdAt", descending: true)
        return AsyncStream(bufferingPolicy: .bufferingNewest(1)) { continuation in
            let listener = query.addSnapshotListener { snapshot, _ in
                guard let snapshot else { return }
                continuation.yield(snapshot.documents.compactMap { try? $0.data(as: Category.self) })
            }
            continuation.onTermination = { _ in listener.remove() }
        }
    }
```

> `uid` and `categoriesCollection` are existing members. `uid` is evaluated once at stream-construction time (the caller only starts streams while signed in).

### Task 2: StuffViewModel live sync + ContentView wiring

**Files:**
- Modify: `MyStuff/ViewModels/StuffViewModel.swift`
- Modify: `MyStuff/Views/ContentView.swift`

**Interfaces:**
- Consumes: `service.itemsStream()`/`locationsStream()`/`categoriesStream()` (Task 1).
- Produces: `StuffViewModel.startLiveSync()` / `stopLiveSync()`.

- [ ] **Step 1: Add live-sync to `StuffViewModel`**

In `MyStuff/ViewModels/StuffViewModel.swift`, add a stored property near the other `@ObservationIgnored` private state (or after `private let service`):

```swift
    @ObservationIgnored private var syncTasks: [Task<Void, Never>] = []
```

Add these methods right after `loadData()`:

```swift
    /// Start real-time listeners so shared/edited items, locations, and categories update live.
    /// Idempotent; call after `loadData()`. Cancel via `stopLiveSync()`.
    func startLiveSync() {
        guard syncTasks.isEmpty else { return }
        let currentUid = service.currentUserId

        syncTasks.append(Task { [weak self] in
            for await raw in service.itemsStream() {
                guard let self else { return }
                self.items = Self.deduped(Self.migratedPhotoFields(raw).map { Self.migratedSharingFields($0, currentUid: currentUid) })
            }
        })
        syncTasks.append(Task { [weak self] in
            for await raw in service.locationsStream() {
                guard let self else { return }
                self.locations = Self.deduped(raw.map { Self.migratedSharingFields($0, currentUid: currentUid) })
            }
        })
        syncTasks.append(Task { [weak self] in
            for await raw in service.categoriesStream() {
                guard let self else { return }
                self.categories = Self.deduped(raw)
            }
        })
    }

    /// Cancel live-sync tasks (removes the underlying Firestore listeners via stream termination).
    func stopLiveSync() {
        syncTasks.forEach { $0.cancel() }
        syncTasks.removeAll()
    }
```

> `service` is a `let` captured by the `Task`; the `Task` created inside a `@MainActor` method inherits main-actor isolation, so the `self.items = …` mutations run on the main actor. `[weak self]` avoids a retain cycle (the VM owns the tasks). Migration is NOT persisted here — that stays in `loadData()`; the stream only maps for display.


### Task 4: SocialViewModel live sync + ContentView wiring

**Files:**

- [ ] **Step 1: Add live-sync to `SocialViewModel`**

In `MyStuff/ViewModels/SocialViewModel.swift`, add a stored property near `private let service`:

```swift
    @ObservationIgnored private var syncTasks: [Task<Void, Never>] = []
```

Add these methods right after `load()`:

```swift
    /// Start real-time listeners so friends, incoming/outgoing requests, and the badge update live.
    /// Idempotent; call after `load()`. Cancel via `stopLiveSync()`.
    func startLiveSync() {
        guard syncTasks.isEmpty else { return }

        syncTasks.append(Task { [weak self] in
            for await friends in service.friendsStream() {
                guard let self else { return }
                self.friends = friends
            }
        })
        syncTasks.append(Task { [weak self] in
            for await incoming in service.incomingRequestsStream() {
                guard let self else { return }
                self.incomingRequests = incoming.filter { $0.status == .pending }
            }
        })
        syncTasks.append(Task { [weak self] in
            for await outgoing in service.outgoingRequestsStream() {
                guard let self else { return }
                self.outgoingRequests = outgoing.filter { $0.status == .pending }
                await self.reconcileAcceptedOutgoing(outgoing: outgoing, existingFriends: self.friends)
            }
        })
    }

    /// Cancel live-sync tasks (removes the underlying Firestore listeners).
    func stopLiveSync() {
        syncTasks.forEach { $0.cancel() }
        syncTasks.removeAll()
    }
```

> `reconcileAcceptedOutgoing(outgoing:existingFriends:)` already exists (from P1) and is idempotent — when an outgoing request flips to `accepted`, the requester writes its own friend doc, which the friends stream then reflects live.

- [ ] **Step 2: Wire it in `ContentView`**

In `MyStuff/Views/ContentView.swift`, in the existing social `.task` (the one that sets `social.onUnfriend`, awaits `social.load()`, and sets `viewModel.friends = social.friends`), start live sync after the load; and cancel it in the existing `.onDisappear` (added in Task 2):

```swift
        .task {
            social.onUnfriend = { friendUid in
                await viewModel.unshareEverything(withFriend: friendUid)
            }
            await social.load()
            viewModel.friends = social.friends
            social.startLiveSync()
        }
        .onDisappear {
            viewModel.stopLiveSync()
            social.stopLiveSync()
        }
```

> The existing `.onChange(of: social.friends) { viewModel.friends = social.friends }` keeps the StuffViewModel friends mirror current as the friends stream updates — so shared-entity owner labels stay correct live too.

## Known follow-ups (later)
- On sign-out the app swaps to `SignInView` (ContentView disappears → `.onDisappear` cancels the tasks/listeners); if a race left a listener attached it would error `permission-denied` and the callback's `guard let snapshot` ignores it — harmless.
- `loadData()`'s one-time server fetch is now slightly redundant with the stream's first snapshot (one extra read on launch); kept to preserve the existing migration-persist path. Could be consolidated later.
- Streams use `.bufferingNewest(1)`; rapid successive edits collapse to the latest snapshot (correct, since each yield replaces the whole array).
